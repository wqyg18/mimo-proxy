# CLAUDE.md

## 项目概述

MIMO Proxy — 本地 HTTP 代理，管理多个 MIMO API key，自动轮换和冷却。

- 入口：`node index.js`（或 `npm start`）
- 端口：`MIMO_PROXY_PORT`，默认 8080
- 目标：`token-plan-cn.xiaomimimo.com`

## 添加新 Key

用户给出 key 时，可能需要解码处理。处理流程：

### 1. 判断是否需要 base64 解码

如果 key 不以 `tp-` 开头，且看起来像 base64 编码（纯字母数字 + `+/=`），尝试 base64 解码。已有 `tp-` 前缀的跳过此步。

### 2. 去除干扰字符

用户可能在 key 中插入中文字符或无意义片段来混淆。常见模式：

- 插入中文字符（如 删、掉、文、字）作为分隔标记
- 标记之间的内容是干扰串，需要删除

根据用户说明去除对应字符/片段。

### 3. 校验

最终 key 必须满足：

- **前缀**：以 `tp-` 开头
- **长度**：51 个字符（`tp-` + 48 位）
- **字符集**：仅小写字母和数字

校验不通过时，向用户确认后再写入。

### 4. 写入 keys.json

追加到 `keys.json` 数组末尾，保持 JSON 格式正确。
